# 事务与回滚

事务模型用于保护 R2 及以上的写动作。目的不是“尽量写成功”，而是“失败时保持工作区干净且可恢复”。

## 事务生命周期

```text
snapshot -> execute -> verify -> commit
                     \-> rollback
```

## 事务包络

一个可回滚写动作至少需要：

- 目标文件列表
- 执行步骤
- 验证命令或验证指标
- 回滚策略

## R2 写动作规则

1. 先快照，再执行。
2. 验证失败先尝试一次受控修复。
3. 修复仍失败则自动回滚。
4. 回滚结果必须记录到反馈日志。

## 文本补丁规则

`apply_patch_sandboxed` 默认要求：

- `oldText` 唯一匹配
- 匹配不到则拒绝
- 匹配多次也拒绝

这样做是为了避免“补丁打到了不确定位置”。

## 冻结承诺

v4.x LTS 期间，写动作如果不能提供事务、验证和回滚，就不能被当成自动 R2 动作放行。
