# v4.3.17 验收记录

## 版本目标

`v4.3.17` 的工作重点是：

- 事件日志锁的陈旧锁回收（崩溃写入方遗留锁不再冻结事件循环）
- 凭据密钥派生按进程缓存（去掉 `loadCredentials` 的重复 PBKDF2 开销）
- 不放宽任何自动化和安全边界

## 本版确认项

1. 事件日志哈希链在并发追加下仍完整；陈旧锁被回收，新鲜锁不会被抢占（mtime 守卫 + `wx` 原子创建）。

3. 命令执行、文件系统边界、动作执行与插件加载边界均未被放宽。
4. 发布门、设计矩阵、版本号与当前版本一致。

`v4.3.17` 满足当前 release gate，可作为维护发布版本。
